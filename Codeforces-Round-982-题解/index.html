A - Alice’s Adventures in “Chess”

题意

给你一个只包含 “NESW” 的字符串 sss。初始位置在 (0,0)(0, 0)(0,0)，问：如果不断循环 sss 的操作，能否到达一个另一个点 (a,b)(a, b)(a,b)。

分析

不想直接暴力，写了个 O(∣s∣)O(\lvert s \rvert)O(∣s∣) 的解法。记第 iii 次操作的位移向量为 tit_iti​，那么如果能到达 (a,b)(a, b)(a,b)。说明 ∃  x=0,1,2,⋯ ;  y=0,1,2,⋯ ,n−1\exist\;x = 0, 1, 2, \cdots;\; y = 0, 1, 2, \cdots, n-1∃x=0,1,2,⋯;y=0,1,2,⋯,n−1 使得 x∑i=1nti+∑i=1yti=(a,b)x\sum_{i=1}^n t_i + \sum_{i=1}^y t_i = (a, b)x∑i=1n​ti​+∑i=1y​ti​=(a,b)。我们枚举 yyy, 然后通过模长相等是向量相等的必要条件计算出 xxx 的值，然后带入验证 xxx 是否满足上面式子。

B - Alice’s Adventures in Permuting

题意

给你一个等差数列，你需要对它的前 nnn 项做操作。每次操作选择最大的那项，将它变为集合中的 mex 值。问需要多少步变成 0,1,2,⋯ ,n−10, 1, 2, \cdots, n-10,1,2,⋯,n−1

分析

显然如果公差 d≠0d \neq 0d=0 显然是可以的。需要计算不在 [0,n−1][0, n - 1][0,n−1] 范围内的数，输出就行。如果 d=0d = 0d=0，说明全部数都挤成了一坨。如果首项 a<n−2a < n - 2a<n−2，就是不行的。如果 n−2≤a≤n−1n - 2 \leq a \leq n - 1n−2≤a≤n−1，只需要执行 n−1n - 1n−1 次。如果 a>na > na>n 需要执行 nnn 次。

C - Alice’s Adventures in Cutting Cake

题意

给你一个长度为 nnn 的数组 aaa，将 aaa 分成 m+1m + 1m+1 段。要求其中 mmm 段的区间和 sum≥vsum \geq vsum≥v。求剩下那一段的区间和最大为多少。如果不能做到第一个条件，输出 −1-1−1。注意区间的长度可能为 000

分析

一定是左边若干段，右边若干段，然后选取中间的部分。枚举左边的段数计算。预处理：左边选取 iii 段时，至少需要取到左边第几个数。右边选取 iii 段时，至少需要取到右边第几个数。

D - Alice’s Adventures in Cards

题意

你现在的手牌是 111，你需要执行若干次操作，将你的手牌单调递增地替换，直到你的手牌为 nnn。输出方案或报告不可能。给你三个排列 qqq kkk jjj。假设现在的手牌为 xxx，那么 xxx 可以替换成 yyy 当且仅当存在一个排列 p∈{q,k,j}p \in \{q, k, j\}p∈{q,k,j}，使得 px>pyp_x > p_ypx​>py​.

分析

暴力做法就是直接对于每个逆序对连边，然后跑一下dfs得出答案，时间复杂度 O(n2)O(n^2)O(n2)，考虑优化建图，然后想了大半天发现好像怎样也无法突破边数为 O(n2)O(n^2)O(n2) 的限制。看看江莉的做法，从左往右扫，判断当前位置是否可以被获取。判断方法是是否存在一个序列，使得前面可以获得的点的最大值 max⁡pi\max{p_i}maxpi​（iii 是可获取的）大于当前位置的值 pjp_jpj​. 这个最大值初始化为 p1p_1p1​，每次获取一个数时更新即可。这样就可以记录那些数字可以被获取。如果 nnn 可以被获取，那么输出 “YES”，否则输出 “NO”。构造方案就很简单了，只需要在判断时候可以获取的地方加上入边的存储即可。

E - Alice’s Adventures in the Rabbit Hole

题意

Alice 和 Bob 在树上玩游戏，树根为 111，Alice 想要走到根，Bob 想要走到叶子。他们通过抛硬币觉得谁行动。问对于每个节点来说，Alice 的获胜概率（Mod 998244353）

分析

考虑长度为 n+1n+1n+1 的链的情况，pi=12pi−1+12pi+1,i∈[1,n]p_i = \frac{1}{2}p_{i-1} + \frac{1}{2}p_{i+1}, i \in [1, n]pi​=21​pi−1​+21​pi+1​,i∈[1,n]。那么这个东西怎么求呢，借用一下代数芝士，可以知道：

[2−10⋯00−12−1⋯000−12⋯00⋮⋮⋮⋱⋮⋮000⋯2−1000⋯−12]n[p1p2p3⋮pn−1pn]=[000⋮01] \left [ \begin{matrix} 2&-1&0&\cdots &0&0\\ -1&2&-1&\cdots &0&0\\ 0&-1&2&\cdots &0&0\\ \vdots&\vdots&\vdots&\ddots&\vdots&\vdots\\ 0&0&0&\cdots&2&-1\\ 0&0&0&\cdots&-1&2 \end{matrix} \right ]_{n} \left [ \begin{matrix} p_1\\ p_2\\ p_3\\ \vdots\\ p_{n-1}\\ p_{n} \end{matrix} \right ]= \left [ \begin{matrix} 0\\ 0\\ 0\\ \vdots\\ 0\\ 1 \end{matrix} \right ] ⎣⎡​2−10⋮00​−12−1⋮00​0−12⋮00​⋯⋯⋯⋱⋯⋯​000⋮2−1​000⋮−12​⎦⎤​n​⎣⎡​p1​p2​p3​⋮pn−1​pn​​⎦⎤​=⎣⎡​000⋮01​⎦⎤​

然后直接开挂掏出矩阵计算器，计算一下左边这个矩阵的逆。这里只需要关注最后一列就行，因为右边向量只有最有一位是 111。然后我们就可以惊奇的发现 pi=in+1,i∈[0,n+1]p_i = \frac{i}{n+1}, i\in[0, n+1]pi​=n+1i​,i∈[0,n+1].

假如它不是一条链怎么办，那就直接拆开成链。我们让父节点做链头，距离最近的后代叶做链尾。

为什么是后代叶子呢，因为我猜测（我是不会啊我会我肯定说出来的qwq）由于父节点的获胜概率更大，所以聪明的 Bob 一定不会选择走距离更近的其他叶子的方向，因为这样需要经过父节点。

为什么是父节点而不是直接选取根节点呢，考虑父节点 uuu 节外生叶的情况，此时父节点的概率一定不是根据 vvv 的后代叶子来计算的，而是根据 vvv 的兄弟叶子节点（节外生出的那个叶）来计算的，所以上面的矩阵就不成立了。

注意到，这个时候链头的概率就不为 111 了，但是没关系，只是右边向量的 111 变了一下而已，此时概率等于 pi=pn+1in+1,i∈[0,n+1]p_i = p_{n+1}\frac{i}{n+1}, i\in[0, n+1]pi​=pn+1​n+1i​,i∈[0,n+1]，pn+1p_{n+1}pn+1​ 表示链头所对应的概率。

所以只需要预处理一下最近的后代叶子的距离，然后就可以bfs/dfs扫一遍整棵树计算答案了。

yy时间

这题似乎可以通过 “短链剖分” 来做，因为每次确定好一条链之后就可以计算出整条链上每个节点的概率。但是由于不需要更新什么数据，时间上没有优势，代码也会很长，所以没有必要这么做。啊这是我猜的，没有验证过。
