A - Preparing for the Olympiad

Tag

贪心

题意

A, B 两人各有一个训练计划，一共有 nnn 天。A 在第 iii 天做 aia_iai​ 题，B 在第 iii 天 bib_ibi​。A 可以任意选择若干天来做题，B 只能在 A 训练的下一天（如果存在）训练。问 A 最多比 B 多做多少道题

分析

对于任意的 1≤i<n1 \leq i \lt n1≤i<n，如果有 ai>bi+1a_i > b_{i + 1}ai​>bi+1​，那么 A 在第 iii 天训练。A 第 nnn 天无条件训练。

1≤n≤1091\leq n \leq 10^91≤n≤109

先计算需要走多少个三天，然后再计算剩下的路程需要走多少天

C - Preparing for the Exam

Tag

模拟

题意

一共有 nnn 个问题，和 mmm 次考试。每个考试会考除了 aia_iai​ 以外的题目。你知道 kkk 道题 b1,b2,⋯ ,bkb_1, b_2, \cdots, b_kb1​,b2​,⋯,bk​ 的答案。对于每场考试输出是否全都会做。

分析

如果 k=nk = nk=n 输出 nnn 个 111，如果 k<n−1k \lt n - 1k<n−1 输出 mmm 个 000。否则判断没有出的那一题是否知道答案，如果知道那就输出 000（因为不会的那一题出了），如果不知道那就输出 111

D - Counting Pairs

Tag

枚举，二分

题意

给你一个数组 aaa，你需要从中删除两个数，使得剩下的数的和处于区间 [x,y][x, y][x,y] 中。问删除的方案。

1≤n≤2⋅1051≤x<y≤2⋅10141\leq n \leq 2\cdot 10^5\\1\leq x \lt y \leq 2 \cdot 10^{14}1≤n≤2⋅1051≤x<y≤2⋅1014.

分析

这是一个非常经典的问题：查找满足某个条件的二维数对 (i,j)(i, j)(i,j) 的个数。枚举 iii，查找 jjj。由于这个数组的顺序没什么用处，所以先对这个数组排序。枚举第一个删除的数，然后在第一个数的后面查找处于区间 [sum−ai−r,sum−ai−l][sum - a_i - r, sum - a_i - l][sum−ai​−r,sum−ai​−l] 的数的个数，直接二分就行了。二分的区间要是从 i+1i + 1i+1 开始，以免选到两个下标相同的数。

E - Best Price

Tag

贪心，枚举

题意

你有 111 种商品和 nnn 个顾客。每个顾客有两个预期价值 aia_iai​ bib_ibi​。如果商品价格 x≤aix \leq a_ix≤ai​，那么顾客会购买。如果 ai<x≤bia_i \lt x \leq b_iai​<x≤bi​，那么顾客会购买并留下差评。如果 x>bix \gt b_ix>bi​，那么顾客不会购买（也没有差评）。

你需要选择 xxx，使得收益最大，限制差评数量不能超过 kkk 条。

1≤n≤2⋅1050≤k≤n1≤ai<bi≤2⋅1091\leq n \leq 2\cdot 10^ 5\\0\leq k \leq n\\1\leq a_i \lt b_i \leq 2 \cdot 10^91≤n≤2⋅1050≤k≤n1≤ai​<bi​≤2⋅109

分析

一个比较显然的结论是：xxx 一定是 aaa 中的数或 bbb 中的数。因为如果不是，那么你可以继续抬高价格直到 xxx 是 aaa 或 bbb 中的数，而且这个太高价格是没有代价的。

所以就枚举价格就行了，枚举价格的同时维护两个值：购买的人数和差评的人数。具体的做法是使用map记录每个数在 aaa 中出现了多少次，在 bbb 中出现了多少次。枚举时先记录答案在更新人数。

F - Joker

Tag

思维

题意

你有一副牌，牌号为 1,2,⋯ ,n1, 2, \cdots, n1,2,⋯,n。其中 Joker 牌在第 mmm 张。你有 qqq 次操作，每次操作将第 aia_iai​ 张牌随机移动到开头或者结尾。问最终 Joker 可能在哪几个位置？

分析

注意到 Joker 只有可能在开头结尾和中间，而且构成一个区间。选择一个位置放到开头结尾，分成三种情况：选到了区间内部，选到了区间左边，选到了区间右边。枚举一下 777 种情况就能证明完成了。然后实际的操作也是根据 777 种情况维护区间的。

G - Snakes

Tag

状压dp

题意

你有一个长度为 10910^9109 的条形游戏场地。你有 nnn 条蛇，蛇的初始长度为 111。你有 mmm 次操作，每次操作选择第 aia_iai​ 条蛇，使其右端点加一或左端点加一。蛇不能碰到另一条蛇。你需要合理选择蛇的初始位置，使得第一条蛇到达的最远距离最小。

1≤n≤201≤q≤2⋅1051\leq n \leq 20\\1\leq q \leq 2 \cdot 10^51≤n≤201≤q≤2⋅105

分析

n=20n = 20n=20 容易想到是状压dp，但是具体怎么状压呢。排列问题暴力做是 O(n!)O(n!)O(n!)，状压可以 O(n⋅2n)O(n\cdot 2^n)O(n⋅2n) 解决。dpijdp_{ij}dpij​ 表示选取情况为 iii，最右边的蛇的编号是 jjj 的情况下的初始最远距离。初始最远距离就是在操作之前的最远距离，也就是最靠右边的蛇所放置的位置。状态转移方程是 dpix=dpjy+gapyx+1dp_{ix} = dp_{jy} + gap_{yx} + 1dpix​=dpjy​+gapyx​+1。其中 j=ixor⁡1<<(x−1)j = i \operatorname{xor} 1 << (x - 1)j=ixor1<<(x−1)，gapyxgap_{yx}gapyx​ 表示如果第 yyy 条蛇跟在第 xxx 条蛇的蛇尾，他们之间需要多少间隔。 gapgapgap 可以通过预处理计算，时间复杂度 O(n2q)O(n^2q)O(n2q)。dp的时间复杂度 O(n2⋅2n)O(n^2\cdot2^n)O(n2⋅2n)
