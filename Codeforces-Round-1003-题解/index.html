C - Skibidus and Fanum Tax

Tag

贪心、二分

题意

给你两个数组 aaa 和 bbb。其中 aaa 长度为 nnn，bbb 长度为 mmm。对于每一个 i∈[1,n]i \in [1, n]i∈[1,n]，最多可以执行一次操作，问是否可以让 aaa 单调不减。

操作：选择一个 j∈[1,m]j \in [1, m]j∈[1,m]，将 aia_iai​ 设置为 bj−aib_j - a_ibj​−ai​

分析

显然是贪心嘛，在满足大于等于前面的数的情况下，让 aia_iai​ 尽可能小。aia_iai​ 的可能取值为 aia_iai​ 和 bj−aib_j - a_ibj​−ai​。第一个取值没什么讲究，第二个取值就可以用到贪心，因为需要 bj−ai>=ai−1b_j - a_i >= a_{i - 1}bj​−ai​>=ai−1​ 那么就有 bj>=ai+ai−1b_j >= a_i + a_{i - 1}bj​>=ai​+ai−1​ 为了使 bj−aib_j - a_ibj​−ai​ 尽可能小，bjb_jbj​ 需要尽可能小，找这个 bjb_jbj​ 可以二分。

D - Skibidus and Sigma

Tag

贪心

题意

ttt 是一个数组，令 f(t)=∑i=1n∑j=1itif(t) = \sum\limits_{i = 1}^n\sum\limits_{j = 1}^i t_if(t)=i=1∑n​j=1∑i​ti​。给你 nnn 个数组，每个数组的长度为 mmm，你需要将 nnn 个数组拼接，也就是 b=ap1+ap2+⋯+apnb = a_{p_1} + a_{p_2} + \cdots + a_{p_n}b=ap1​​+ap2​​+⋯+apn​​ 其中 ppp 为一个排列。求出 f(b)f(b)f(b) 的最小值。

分析

把 f(t)f(t)f(t) 化简一下就是 ∑i=1n(n−i+1)⋅ti\sum\limits_{i = 1}^n (n - i + 1)\cdot t_ii=1∑n​(n−i+1)⋅ti​。所以可以写出这样的三角形：

t1t2⋯tn−1tnt1t2⋯tn−1⋮⋮t1t2t1 \begin{matrix} t_1&t_2&\cdots&t_{n-1}&t_n\\ t_1&t_2&\cdots&t_{n-1}\\ \vdots&\vdots\\ t_1&t_2\\ t_1 \end{matrix} t1​t1​⋮t1​t1​​t2​t2​⋮t2​​⋯⋯tn−1​tn−1​tn​

那么拼接起来的数组也可以类似地写出一个三角形。因为这个写起来比较麻烦，我就不写了。总之你就会发现 f(b)=∑i=1nf(ai)+∑i=1n((n−i)⋅m⋅∑api)f(b) = \sum\limits_{i = 1}^nf(a_i)+\sum\limits_{i = 1}^n ((n-i)\cdot m \cdot \sum a_{p_i})f(b)=i=1∑n​f(ai​)+i=1∑n​((n−i)⋅m⋅∑api​​)

第一个部分是不变的，而且可以在 O(nm)O(nm)O(nm) 的时间复杂度内求出，第二部分可以先求出每个数组 aia_iai​ 的和 ∑ai\sum a_i∑ai​，然后按照贪心，越大的 ∑ai\sum a_i∑ai​ 的系数越大。

E - Skibidus and Rizz

Tag

构造、思维

题意

ttt 是一个二进制串，令 xxx 表示 ttt 中 000 的个数，yyy 表示中 111 的个数，则 f(t)=max⁡(x−y,y−x)f(t) = \max(x - y, y - x)f(t)=max(x−y,y−x)。给你三个整数 n,m,kn, m, kn,m,k 构造一个字符串 sss，使得：000 的个数为 nnn，111 的个数为 mmm。对于所有子串 ttt，max⁡f(t)=k\max f(t) = kmaxf(t)=k。

分析

我们暂且用一下dp的思想，考虑对于一个以 sis_isi​ 结尾的字符串，x−yx - yx−y 的最小值为 dp(i,0)dp(i, 0)dp(i,0)，最大值为 dp(i,1)dp(i, 1)dp(i,1) 那么显然会有递推：

dp(i,0)=dp(i−1,0)±1dp(i,1)=dp(i−1,1)±1 dp(i, 0) = dp(i - 1, 0) \pm 1\\ dp(i, 1) = dp(i - 1, 1) \pm 1 dp(i,0)=dp(i−1,0)±1dp(i,1)=dp(i−1,1)±1

这里的加减 111 主要是取决于 sis_isi​ 是 000 还是 111。所以不难发现如果前面的子串满足 f(t)≤kf(t) \leq kf(t)≤k，那么后面的字符串不断 000 111 交替即可满足 f(t)≤kf(t) \leq kf(t)≤k。

那么最大值为 kkk 怎么办，最简单的想法是在最开始就放 kkk 个相同的字符即可。所以最终的构造方案就是：先在一开始放 kkk 个相同字符，然后接着把 000 和 111 的数量对齐，最后放 010101。其他情况都是输出 −1-1−1，其中有些细节反正写了就知道了。

F - Skibidus and Slay

Tag

思维、模拟

题意

一个长度为 nnn 的序列的众数被定义为出现次数严格大于 ⌊n2⌋\lfloor\frac{n}{2}\rfloor⌊2n​⌋ 的数字。给你一棵树，树有点权，点权范围是 [1,n][1, n][1,n]，问哪些数字 xxx 满足：存在一条长度大于等于 222 的简单路径，使得这条路径上的点权序列的众数。

分析

一个非常非常典的结论是：如果一个序列的众数是 xxx，那么一定可以找到其中的一个连续子序列，它的众数为 xxx 且长度为 222 或 333。也就是 xxxxxx 型和 xyxxyxxyx 型。

对于第一种类型，只需要找相邻的点，判断其点权是否和当前的点点权相同即可。对于第二种类型，我们枚举节点，让这个节点作为 yyy，判断是否存在两个相邻的节点，它们的点权相同。

分析

lcm⁡(x,y)\operatorname{lcm}(x, y)lcm(x,y) 是半素数只有可能是：一个数是半素数，另一个数是半素数的因子；或者两个数为不相等的素数。对于 xxx 是素数，需要另一个数是素数。对于 xxx 是半素数，需要另一个数是其因子。所以解法是枚举 xxx，计算 yyy，非常经典的思路

为什么 xxx 是素数的情况下不能选取 yyy 为半素数：为了防止重复。

H - Skibidus and Capping

Tag

数据结构、组合数学

题意

sss 为一个二进制串，定义 f(s)f(s)f(s) 为 sss 的极大 010101 块的数量。例如 001100001100001100 的答案就是 333，因为有三块，分别是 000000 111111 000000

给你一个二进制串 sss，你有 qqq 次操作，每次操作将 sps_psp​ 反转，然后你需要计算 ∑f(t)\sum f(t)∑f(t) 其中 ttt 为 sss 的子序列（不连续）。

分析

考虑将一个 sps_psp​ 反转，不妨假设 sp=0s_p = 0sp​=0，为了方便区分，下面用 xxx 表示 sps_psp​。考虑哪些子串 ttt 的 f(t)f(t)f(t) 值会发生变化。

  • 不包含 sps_psp​ 的子序列不会改变
  • ⋯0x0⋯\cdots 0x0\cdots⋯0x0⋯ 的子序列 f(t)f(t)f(t) 会增加 222
  • ⋯1x1⋯\cdots 1x1\cdots⋯1x1⋯ 的子序列 f(t)f(t)f(t) 会减少 222
  • ⋯1x\cdots 1x⋯1x 的子序列 f(t)f(t)f(t) 会减少 111
  • ⋯0x\cdots 0x⋯0x 的子序列 f(t)f(t)f(t) 会增加 111
  • x1⋯x1\cdotsx1⋯ 的子序列 f(t)f(t)f(t) 会减少 111
  • x0⋯x0\cdotsx0⋯ 的子序列 f(t)f(t)f(t) 会增加 111

所以只需要维护在 ppp 前面有多少个 ⋯0\cdots 0⋯0 多少个 ⋯1\cdots 1⋯1 在 ppp 后面有多少个 0⋯0 \cdots0⋯ 多少个 1⋯1\cdots1⋯。这部分可以使用线段树来维护。

我们让初始字符串为 00⋯000\cdots000⋯0 然后通过修改得到 sss，总计 n+qn + qn+q 次修改，每次修改的时间复杂度是 O(log⁡n)O(\log n)O(logn) 总共 O((n+q)log⁡n)O((n+q)\log n)O((n+q)logn) 的时间复杂度
