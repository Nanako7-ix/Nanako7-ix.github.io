B - Subsequence Update

Tag

贪心

题意

给你一个长度为 nnn 的数组 aaa 和一个区间 [l,r][l, r][l,r]。你需要执行下面这个操作 111 次，使得 ∑i=lrai\sum_{i = l}^r a_i∑i=lr​ai​ 最大。操作内容为：选择一个子序列，然后反转这个子序列。

分析

反转一个子序列可以视作选取 kkk 对数字，然后各自交换。可以发现对答案有影响的操作是某个区间外面的数和某个区间里面的数交换。所以我们只考虑区间内部和区间外部互换的情况。容易发现这个互换只能是这种情况：选择区间内 kkk 个数字，与区间 [1,l)[1, l)[1,l) 或者区间 (r,n](r, n](r,n] 中的 kkk 个数字互换。

直接分类计算这两种情况即可，每种情况单独贪心选取

C - Remove Exactly Two

Tag

暴力、分类讨论（似乎写麻烦了）

题意

给你一棵树（n=2⋅105n = 2\cdot10^5n=2⋅105），你需要严格删除 222 个节点及其相邻的边，使得操作后的连通块个数最多。

分析

考虑只删除一个点的情况，显然是删除度数最大的那个点，答案即为度数。现在考虑删除两个点的情况，注意到删除第一个节点的时候，会导致第二个节点的度数减少而导致第二步操作的收益下降。

假如度数最大的那个点的度数是小于三次的，那么直接删除这个节点。因为如果你放弃这个节点，那么你第一步的收益就至少减少了 111，而第一步操作导致第二步操作下降的收益最多只有 111，这种情况下是没有必要为了第二步操作而选择放弃第一步选择最大的那个的。

如果出现一次，那么选择只有一个。直接选择度数最大的那个节点删除，然后再找一个度数最大的点。如果度数最大的节点为两个，那么就两个都试一下，答案取最大。

假如度数最大的那个点的度数出现了至少三次，那么一定可以选取两次最大的那个度数。使用反证法，假如不可以，说明每个点都与另外的那几个点存在连边，也就是构成 kkk 阶完全图，在 k≥3k \geq 3k≥3 的情况下是不可能发生的，因为这是树。

分析

先求 kkk，kkk 实际上非常好求，这是一个配对的问题，很显然，只要数量不是压倒性的，就一定可以完美配对。所以 k=min⁡(n,m,n+m3)k = \min(n, m, \frac{n + m}{3})k=min(n,m,3n+m​)

对于一次操作，每次选择删除 aaa 中的两个点和 bbb 中的一个点，或者删除 bbb 中的两个点和 aaa 中的一个点，由于这两种情况是对称的，我们只考虑第一种情况。假设 aaa 中的两个点为 (ax,2)(a_x, 2)(ax​,2) 和 (ay,2)(a_y, 2)(ay​,2)，这个时候发现三角形的面积为 ∣ax−ay∣|a_x - a_y|∣ax​−ay​∣。那么贪心地执行这一次操作，一定是选择 aaa 中最大和最小的元素。

另外为了方便描述，我们称第一种情况为 A 型三角形，第二种情况为 B 型三角形。

考虑在 f(i)f(i)f(i) 到 f(i+1)f(i + 1)f(i+1) 的转移，如果此时可以直接执行一次操作），那么就按照上文所说的贪心地选择最大最小的元素。同时删除另一个数组中的一个值，注意这里你不需要确定需要删除哪一个值，你只需要确定另一数组中可选择的数的数量减少一个即可。

如果你不能直接选择一个三角形，如果 i≠ki \neq ki=k，否则一定是存在一个数组剩余不低于 333 个点而另外一个数组的数字全部消耗。不妨假设 aaa 剩余不低于 333 个点而 bbb 剩余 000 个点。我们可以通过撤销一次先前的 B 型三角形，这样 bbb 中剩余 222 个点，而 aaa 中剩余不低于 444 个点，这样可以构造两个 A 型三角形。这个操作同样是贪心地删除贡献最小的 B 型三角形和添加 222 个贡献最大的 A 型三角形。

只需要双指针维护 aaa bbb 数组中下一步需要选择的节点，并且记录需要额外删除的节点即可，总体还是非常暴力的。
