A - Line Breaks

题意

给你 nnn 个字符串和一个能写的下 mmm 个字符的一张纸。你需要在这张纸上按顺序写这 nnn 个字符串。问你最多能写多少个字符串。

分析

记录一下前 kkk 个字符串有多少个字符，找到第一个大于 mmm 的位置，然后输出减一即可

B - Transfusion

题意

给你一个数组，你可以选择一个 1<i<n1 < i < n1<i<n 进行若干次操作。每次操作为 ai+1a_{i + 1}ai+1​ 加一 与 ai−1a_{i - 1}ai−1​ 减一 或 ai+1a_{i + 1}ai+1​ 减一 与 ai−1a_{i - 1}ai−1​ 加一。问能不能让整个序列相同。

分析

显然，每次操作只会改变相同奇偶位置的值。那么考虑把 aaa 拆成两个部分，对于其中一个部分，可以选择给左边匀一个 111 或者给右边匀一个 111。如果这个序列能变成相同，说明 sum≡0(modcnt)sum \equiv 0 \pmod {cnt}sum≡0(modcnt)。所以只需分奇偶计算是否能整除，以及整除出来的数是否相同即可。

C - Uninteresting Number

题意

给你一个 10510^5105 位的大整数，你可以执行以下操作若干次：选择其中一位数 xxx，将其从 xxx 变成 x2x^2x2。其中要求 x2x^2x2 也要是一位数。问这个数最终能不能变成 999 的倍数。

分析

首先容易想到，只有 1,2,31, 2, 31,2,3 这三个数是可以执行操作的，而且对 111 执行操作是没有意义的。第二个结论是，如果一个数是 999 的倍数，那么每一位的和要求是 999 的倍数。对 222 操作会让这个和加 222，对 333 操作会让这个和加 666。所以枚举选择操作的 333 的个数，然后计算当前的 222 能不能凑够 999 的倍数即可。

D - Digital string maximization

题意

给你一个由 000 到 999 组成的字符串，你可以执行以下操作若干次：选择一个大于 111 的数，与前一位数交换，并且这个数减一。例如 64→3664 \rightarrow 3664→36。问这个字符串最终能形成的字典序最大的串。

分析

显然你需要从前往后贪心地考虑每个位置最大能放多少。而最大能放的数一定是从长度为 101010 的子串中取得。所以你每次选取一个子串，然后扫一遍这个子串找一下 substrj−j\text{substr}_j - jsubstrj​−j 的最大值就行了。时间复杂度 O(10n)O(10n)O(10n)

题意

给你三个字符串 a,b,ca, b, ca,b,c，你需要用 a,ba, ba,b 来生成 ccc，ccc 初始为空。生成方式如下：选择 aaa 或 bbb，取第一个字符，插入到 ccc 的末尾，同时再删除第一个字符。直到两个字符串为空。最后选择 ccc 中的若干个字符，将其替换为任意字符。问至少需要替换多少个字符。数据范围 ∑∣a∣≤2⋅103,∑∣b∣≤2⋅103\sum \lvert a\rvert \leq 2\cdot 10^3, \sum \lvert b\rvert \leq 2\cdot 10^3∑∣a∣≤2⋅103,∑∣b∣≤2⋅103 保证 ∣a∣+∣b∣=∣c∣\lvert a\rvert + \lvert b\rvert = \lvert c\rvert∣a∣+∣b∣=∣c∣

F - Maximum modulo equality

题意

给你一个长度为 nnn 的数组和 qqq 次查询，每次查询给你一个区间 [l,r][l, r][l,r]，问使得 al≡al+1≡al+2≡⋯≡ar(modp)a_l \equiv a_{l + 1} \equiv a_{l+2} \equiv \cdots \equiv a_r\pmod pal​≡al+1​≡al+2​≡⋯≡ar​(modp) 的最大的 ppp。如果 ppp 可以为无穷大，输出 000。

1≤n,q≤2⋅1051\leq n, q \leq 2\cdot 10^51≤n,q≤2⋅105

分析

考虑两个数的情况 x=k1p+t,y=k2p+tx = k_1p+t, y = k_2p+tx=k1​p+t,y=k2​p+t，则 ∣x−y∣=∣k1−k2∣p\lvert x-y\rvert = \lvert k_1-k_2\rvert p∣x−y∣=∣k1​−k2​∣p，也就是说如果两个数对 ppp 同余，那么他们的差一定是 ppp 的倍数。

对于两个数来说，ppp 的最大值就是两个数的差。对于若干个数来说，ppp 的最大值就是任意两个数之间差值的gcd。实际上，这等于 gcd⁡(∣ar−ar−1∣,∣ar−1−ar−2∣,⋯∣al+1−al∣)\gcd(\lvert a_r - a_{r-1}\rvert, \lvert a_{r - 1} - a_{r-2}\rvert, \cdots \lvert a_{l + 1}-a_l\rvert)gcd(∣ar​−ar−1​∣,∣ar−1​−ar−2​∣,⋯∣al+1​−al​∣)。区间查询gcd，可以用ST表。

G - Tree Destruction

题意

给你一棵树，你需要选择树上的一条路径，删除它，这棵树将会分成若干个连通块。你需要计算最多获得多少个连通块。

分析

一眼看过去非常地树形dp。dp[u][0] 表示 uuu 不删除的情况下，uuu 的子树获得的连通块个数。dp[u][1] 表示 uuu 被删除且最多删除一个儿子的情况下，uuu 的子树获得的连通块个数。dp[u][1] 表示 uuu 被删除且最多删除两个儿子的情况下，uuu 的子树获得的连通块个数。

dp[u][0] 的转移：选择一个儿子 vvv，其贡献为 max(dp[v][0] - 1, dp[v][1], dp[v][2]). dp[u][0] 就是 111 加上最大的那个贡献
dp[u][1] 的转移：选择一个儿子 vvv，其贡献为 max(1, dp[v][1]). dp[u][1] 就是 son[u] - 1 加上最大的那个贡献
dp[u][2] 的转移：选择一个儿子 vvv，其贡献为 max(1, dp[v][1]). dp[u][2] 就是 son[u] - 2 加上最大的两个贡献
