A - Brogramming Contest

Tag

签到

题意

给你一个 010101 字符串 sss 和一个空字符串 ttt，你需要执行以下操作，使得 sss 只包含 000，ttt 只包含 111。操作1：选取 sss 的一个后缀，将其接到 ttt 的末尾，并从 sss 中删除。操作2：选取 ttt 的一个后缀，将其接到 sss 的末尾，并从 ttt 中删除。问至少需要多少次操作。

分析

这个操作过程还是很容易分析出来的。选择 sss 中第一个 111 的位置，把它接到 ttt。然后再从 ttt 中找到第一个 000，把它接到 sss。以此循环，就可以得到最佳的操作。需要多少次操作呢，容易发现就是按 010101 分段的段数减 111。段数的计算没什么好说的。

B - Variety is Discouraged

Tag

签到

题意

定义一个数组 aaa 的分数为数组的长度减去 aaa 中不同元素的个数。你可以执行最多一次操作：选取一个区间 [l,r][l, r][l,r]，删除 al,al+1,⋯ ,ara_l, a_{l + 1}, \cdots, a_ral​,al+1​,⋯,ar​。你需要使得操作之后 aaa 的分数尽可能高，并且 aaa 的长度尽可能短。输出你执行的操作。如果不操作输出 000，如果存在多个答案输出任意一个。

分析

这个分数用了一个减法，考虑把这个分数的表示转化以下，容易发现 aaa 的分数等于 aaa 中重复元素的个数。所以如果你删除一个只出现过一次的数字，那么是不会改变分数的，如果你删除的数字在数组中出现的次数大于等于 222，那么这个数删除之后数组的得分一定会减少。因为你可以不执行操作，这种情况下 aaa 的分数不会减少。所以为了让操作后 aaa 的分数尽可能高，你只能删除仅出现过一次的数组。使用 010101 表示一个位置是否可以删除。找到最长的连续的 111，这个就是需要删除的部分。如果不存在 111，那么就是不执行操作。

C - Remove the Ends

Tag

模拟

题意

给你一个非 000 整数的数组 aaa，你可以执行任意次以下操作。你的最高得分多少（初始分数为 000）。操作：选择一个 iii，你获得 ∣ai∣|a_i|∣ai​∣ 分。如果 ai<0a_i < 0ai​<0，那么删除 ai+1,ai+2,⋯ ,ana_{i + 1}, a_{i + 2}, \cdots, a_nai+1​,ai+2​,⋯,an​。如果 ai》0a_i 》 0ai​》0，那么删除 a1,a2,⋯ ,ai−1a_1, a_2, \cdots, a_{i - 1}a1​,a2​,⋯,ai−1​。

分析

假如你需要拿一个小于 000 的数 aia_iai​，那么在它后面的大于 000 的数都不能拿，因为你拿了这些数之后，你就不能拿 aia_iai​ 了。而在它后面的小于 000 的数字都可以拿，而且是没有任何代价地拿。因为反正你后面的正数都不能拿了，那就按照从后往前的顺序拿负数直到 aia_iai​。考虑枚举你拿的最靠前的负数的位置，后面的负数可一个随便拿，而前面的正数也是可以随便拿的。所以答案就是 aia_iai​ 前面的正数的贡献加上 aia_iai​ 后面负数的贡献。

枚举的时候先统计拿所有负数的答案，然后按顺序枚举 iii，如果是正数就更新正数的贡献，负数就更新负数的贡献。更新负数的贡献之前先计算一下答案，表示刚好最靠前的负数选在这里的时候的答案。然后最后再加上全拿正数的贡献就是最终的答案。

Tag

数据结构

题意

给你一个长度为 nnn 的数组 aaa，给你 qqq 个查询，每个查询给你一个整数 xxx，问你最多执行多少次以下操作：选择 aaa 的最后一个数字 ana_nan​，如果 an≤xa_n \leq xan​≤x，那么删除掉 ana_nan​，并且 xxx 变成 x⊕anx \oplus a_nx⊕an​。

分析

一个约定：用 std::__lg⁡(x)\operatorname{std::\_\_lg}(x)std::__lg(x) 表示 xxx 的位数，例如 ai=(1011)2a_i = (1011)_2ai​=(1011)2​ 的最高位是 333

第一种解法，考虑一次查询。注意到 xxx 异或上一个比 xxx 小的数的时候，最高位只减不增。所以考虑维护 aia_iai​ 的最高位，每次 xxx 往前合并的时候，找到从右往左数第一个最高位大于等于 xxx 最高位的位置，判断是否能合并。如果这个位置的最高位和 xxx 的最高位相等，那么 xxx 的最高位一定会减小，如果大于 xxx 的最高位，那么直接停止操作。由于 xxx 的最高位最多减少 303030 次，所以时间复杂度是 O(30log⁡n)O(30\log n)O(30logn)。那个 logloglog 是用来找从右往左数第一个最高位大于等于 xxx 最高位的位置的。我的解法是使用线段树上二分。

第二种解法，直接滑板火箭启动，开一个 01Trie，把所有的查询放到一起来操作。考虑对 aia_iai​ 执行操作，把所有比 aia_iai​ 小的数删掉，同时记录它们是死在 aia_iai​ 前的。其他的数由于可以合并 aia_iai​，所以全局异或一个 aia_iai​。全局异或使用懒标记（lzy_tag）记录即可，我的懒标记为了图方便需要用 O(H)O(H)O(H) 的时间复杂度初始化，将整个二进制数reverse一下，如果维护一下高度的话就只需要 O(1)O(1)O(1) 的时间复杂度就能全局异或。

E - Mycraft Sand Sort

Tag

组合数学，二分，数据结构

题意

给你一个长度为 nnn 的排列 ppp 和一个长度为 nnn 的颜色数组 ccc。其中 nnn 的范围 2⋅1052\cdot 10^52⋅105，颜色数组的值域为 [1,n][1, n][1,n]。初始你会得到这样的一个网格：第 iii 行放置了 pip_ipi​ 个颜色为 cic_ici​ 的方块。这些方块是靠左的，也就是第 jjj 个方块是放在第 jjj 列的，中间不存在间隙。然后你需要对这个网格进行重力排序，就是给整个网格添加一个向下的重力，下面没有方块的方块会往下掉落，具体可以看这个图：

现在问你存在多少个 (p′,c′)(p', c')(p′,c′) 使得它们进行重力排序之后的结果和 (p,c)(p, c)(p,c) 的结果相同。模数为 998244353998244353998244353。

分析

对于任意的 (p,c)(p, c)(p,c)，图形的形状是不会发生变化的，也就是第一列有 nnn 个方格，第二列有 n−1n - 1n−1 个方格，以此类推。同时发现进行重力排序之后，图形的第一列的颜色是不会发生变化的，它的颜色和 ccc 中的颜色相同，所以可以得出 c′=cc' = cc′=c。那么我们的任务就是确定 p′p'p′ 的数量。

这个有颜色的问题一般会先按颜色分类考虑。考虑某一种颜色，发现它所对应的集合是不会变化的，例如上图中绿色对应的行，它们的长度为 1，21， 21，2，那么 p′p'p′ 中绿色对应的行也应该是 1,21, 21,2。证明起来很简单，找到最大值，表示 p′p'p′ 中这种颜色对应的最大值也是这个数，然后把最大值去掉考虑剩下的部分，这样可以递归地找最大值。所以可以得出这两个集合是相等的。

我们把 ppp 中相同颜色对应的 pip_ipi​ 拿出来，我们需要做的是对他们排列。那么交换两个数的条件是什么，容易发现上图中绿色的顺序是不能交换的，也就是不能上面放一个绿色下面放两个绿色。加入改一下这个图，把 ppp 变成 [4,3,1,2,5][4, 3, 1, 2, 5][4,3,1,2,5]，你会发现这个绿色就可以互换了。你大概可以发现：如果两个数字之间可以交换，那么这两个数字中间其他颜色的最大值需要小于等于这两个数字。

所以你可以想到，对于任意一个数字，它的可交换区间都是确定的，也就是区间内不能出现比这个数字大的其他颜色的数字。你发现这个约束条件是具有单调性的，所以你可以通过二分计算这个区间。二分的check需要查询区间最大值，这里使用线段树。不使用 ST 表是因为你需要计算的是其他颜色的最大值，所以在二分之前需要把当前颜色去掉，二分结束后需要把当前颜色加上，即需要带修。

下面还有一个结论，就是你所确定的这些区间要么是不交的，要么是包含关系。这个的证明其实并不难，假如两个区间 [l1,r1][l_1, r_1][l1​,r1​] 和 [l2,r2][l_2, r_2][l2​,r2​] 满足 l1<l2<r1<r2l_1 < l_2 < r_1 < r_2l1​<l2​<r1​<r2​，由 r2>r1r_2 > r_1r2​>r1​ 可以得出确定第二个区间的数一定大于第一个区间的数，但是 l1<l2l_1 < l_2l1​<l2​ 却得出了相反的结论。

接下来你需要做的就是，对于每一个数字都对应一个区间，表示这个数字这只能在这个区间中，你需要计算总共有多少种方案。对于一个数，他有一个区间，如果该区间内部不存在其他区间，那么整个区间都可以放置这个数，也就是方案为区间长度。否则就是区间长度减去内部的区间的个数。

按照先处理小区间再处理大区间的方式做就行了。保证这个处理顺序，需要按右端点排序，这样后处理的区间要么和前面的区间不相交，要么是前面区间的超集。计算该区间的子区间的个数，只需要计算之前处理的区间中，有多少个区间的左端点 lll 大于等于当前区间的左端点。这个可以使用平衡树做。
