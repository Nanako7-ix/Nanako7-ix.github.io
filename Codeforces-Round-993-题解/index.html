B - Normal Problem

题意

一家商店的玻璃橱窗上画着一个字符串，其中只有字符 “p”、”q “和 “w”。A 走过商店，站在玻璃橱窗正前方，观察到字符串 aaa 。然后，B 走进商店，直视同一个玻璃橱窗，观察到字符串 bbb 。

给你字符串 aaa。输出 bbb。

分析

题目的意思是说，给你一个字符串，然后你从字符串的背面看，这个字符串是什么样的。结合样例可以发现，你需要将字符串reverse一下，然后 ‘p’ 和 ‘q’ 互换

C - Hard Problem

题意

教室里的座位排成 222 行，每行有 mmm 个座位。

a+b+ca + b + ca+b+c 只猴子在上课。有 aaa 猴子只想坐在第 111 排， bbb 只想坐在第 222 排，剩下的 ccc 只说他没意见。每个座位只能坐一只猴子。问最多可以让多少只猴子坐下。

分析

第一种猴子最多有 min⁡(a,m)\min(a, m)min(a,m) 个有座位坐，第二种猴子最多有 min⁡(b,m)\min(b, m)min(b,m) 个有座位坐。剩下的座位给第三种猴子坐。

D - Harder Problem

题意

给定一个正整数序列，如果一个数出现的次数最多，那么这个正整数就称为序列的模。例如，[2,2,3][2,2,3][2,2,3] 的模是 222。序列 [9,9,8,8,7,7][9,9,8,8,7,7][9,9,8,8,7,7] 的模是 999 、888 和 777。

给你一个长度为 nnn 的数组 aaa 。你需要构造一个长度为 nnn 的数组 bbb，使得对任意 i∈[1,n]i\in[1, n]i∈[1,n] 有：aia_iai​ 是序列 [b1,b2,…,bi][b_1, b_2, \ldots, b_i][b1​,b2​,…,bi​] 的模，且 1≤bi≤n1 \leq b_i \leq n1≤bi​≤n。

分析

有一个很重要的性质，一个数组的模可以有很多个。所以让每个数都是模就行了，也就是每个数都出现一次。如果 aia_iai​ 在 bbb 中没有出现，那么有 bi=aib_i = a_ibi​=ai​。反之 bib_ibi​ 是一个在 aaa 中没有出现的数。

F - Easy Demon Problem

题意

对于任意矩阵，Robot 将其 beauty 定义为矩阵中元素的总和。

Robot 给出了一个长度为 nnn 的数组 aaa 和一个长度为 mmm 的数组 bbb。并构造了一个 n×mn \times mn×m 的矩阵 MMM，满足 Mij=ai⋅bjM_{ij}=a_i\cdot b_jMij​=ai​⋅bj​ 。

给出 qqq 个查询，每个查询都包含一个整数 xxx 。对于每个查询，请判断是否可以执行一次下面的操作，从而使 MMM 的 beauty 为 xxx：

  1. 选择整数 rrr 和 ccc，1≤r≤n1 \leq r \leq n1≤r≤n 且 1≤c≤m1 \leq c \leq m1≤c≤m
  2. 对所有 (i,j)(i,j)(i,j) 满足 i=ri=ri=r 或 j=cj=cj=c，设 MijM_{ij}Mij​ 为 000

你每次查询时执行的操作时暂时的，也就是说，在查询过程中，你不会将任何元素设置为 000，你只需要输出是否存在 rrr 和 ccc，使得对 rrr, ccc 执行操作后，矩阵的 beauty 将为 xxx 。

数据范围：

∑n≤2⋅105∑m≤2⋅105∑q≤5⋅104∣x∣≤2⋅105\sum n \leq2\cdot10^5\\\sum m \leq2\cdot10^5\\\sum q \leq5\cdot10^4\\ \lvert x \rvert \leq 2\cdot10^5∑n≤2⋅105∑m≤2⋅105∑q≤5⋅104∣x∣≤2⋅105

分析

这个数据范围还是挺大的，所以考虑计算一下式子。

设 aaa 数组的和为 c0c_0c0​，bbb 数组的和为 c1c_1c1​。假设我删掉第 iii 行和第 jjj 列，那么矩阵的元素和应该变成 c0c1−c1ai−c0bj+aibj=(c0−ai)(c1−bj)c_0c_1 - c_1a_i - c_0b_j + a_ib_j = (c_0 - a_i)(c_1 - b_j)c0​c1​−c1​ai​−c0​bj​+ai​bj​=(c0​−ai​)(c1​−bj​)。令 Ai=c0−aiA_i = c_0 - a_iAi​=c0​−ai​，Bi=c1−biB_i = c_1 - b_iBi​=c1​−bi​

所以对于每次查询，可以转化为是否存在 i,ji, ji,j 使得 AiBj=xA_iB_j = xAi​Bj​=x 成立。由于 AiA_iAi​ 和 BiB_iBi​ 都是整数，所以选择枚举 xxx 的因数，然后查询因数是否存在。

时间复杂度 O(nlog⁡n+qxlog⁡n)O(n\log n + q\sqrt{x}\log n)O(nlogn+qx​logn)

G - Medium Demon Problem

easy version
hard version

题意

一群 nnn 蜘蛛聚在一起交换毛绒玩具。最初，每只蜘蛛都有 111 个毛绒玩具。每年，如果蜘蛛 iii 至少有一个毛绒玩具，他就会给蜘蛛 rir_iri​ 一个毛绒玩具。否则，他什么也不会做。请注意，所有毛绒玩具的转移都是同时进行的。

在 easy version 中，每只蜘蛛会把多余的玩具丢掉（也就是说持有的玩具个数最多为 111）。

在 hard version 中，每只蜘蛛不会丢玩具。

如果每只蜘蛛在本年度(本年度交换前)拥有的毛绒玩具数量与上一年度(上一年度交换前)相同，则本年度的进程是稳定的。请注意， 111 年不可能是稳定的。

请找出进程趋于稳定的第一年。

数据范围：

2≤n≤2⋅1051≤ri≤n2\leq n \leq 2\cdot 10^5\\1 \leq r_i \leq n2≤n≤2⋅1051≤ri​≤n

分析

先把这个输出翻译一下，就是操作 xxx 次后是稳定状态，那么输出 x+2x+2x+2，现在找达到稳定状态需要进行多少次操作

首先需要注意到这是一座基环树森林，和骑士是一样的。
其次注意到，这个环一定是个强连通分量，这也是很容易发现的。
第三注意到，每棵树都是内向基环树，这和第二点其实是很相似的。

那这个过程就很清晰了，边上的树不断向环内转移贡献，如果边上所有的树持有的玩具都是 000，这就是稳定状态了。对于 easy version 来说，贡献的速度取决于深度最大的点的深度。对于 hear version 来说，取决于树的大小。

所以解法是拓扑排序中DP，维护深度和树的大小（已经是最简单的dp了）。值得注意的是，对于 G2 来说：设 uuu 为环上的点，vvv 为 uuu 的儿子，那么你需要计算的是 vvv 的大小而不是 uuu 的大小。因为 uuu 的不同的儿子是可以同时向 uuu 转移贡献的。

时间复杂度 O(n)O(n)O(n)，个人喜欢无向图，所以建的是无向图。当然有向图也是一样的。

H - Hard Demon Problem Problem

题意

Swing 给出了一个包含正整数的 n×nn \times nn×n 矩阵 MMM 。他有 qqq 个查询要问你。

对于每次查询，他都会给出四个整数 x1x_1x1​, y1y_1y1​, x2x_2x2​, y2y_2y2​，并要求你将以 (x1,y1)(x_1, y_1)(x1​,y1​) 和 (x2,y2)(x_2, y_2)(x2​,y2​) 为界的子矩阵平铺成数组 AAA。

查询 ∑i=1∣A∣Ai⋅i\sum_{i=1}^{|A|} A_i \cdot i∑i=1∣A∣​Ai​⋅i 的值(所有 iii 上的 Ai⋅iA_i \cdot iAi​⋅i 之和)。

2≤n≤20002\leq n \leq 20002≤n≤2000 1≤q≤1061\leq q \leq 10^61≤q≤106

分析

byd又是前缀和，其实这题就是推公式就结束了，写个计算过程

答案可以写成这个形式：

ans=∑i=l1r1∑j=l2r2aijkij ans = \sum_{i = l_1}^{r_1}\sum_{j = l_2}^{r_2} a_{ij} k_{ij} ans=i=l1​∑r1​​j=l2​∑r2​​aij​kij​

展开一下 kkk：

k=(i−l1)(r2−l2+1)+(j−l2+1)=(r2−l2+1)i+j−l1(r2−l2+1)−l2+1 \begin{aligned} k &= (i - l_1)(r_2 - l_2 + 1) + (j - l_2 + 1)\\ &= (r_2 - l_2 + 1) i + j - l_1(r_2 - l_2 + 1) - l_2 + 1 \end{aligned} k​=(i−l1​)(r2​−l2​+1)+(j−l2​+1)=(r2​−l2​+1)i+j−l1​(r2​−l2​+1)−l2​+1​

记：

t1=(r2−l2+1)t2=1t3=−l1(r2−l2+1)−l2+1 \begin{aligned} t_1 &= (r_2 - l_2 + 1)\\ t_2 &= 1\\ t_3 &= -l_1(r_2 - l_2 + 1) - l_2 + 1\\ \end{aligned} t1​t2​t3​​=(r2​−l2​+1)=1=−l1​(r2​−l2​+1)−l2​+1​

则：

k=t1  i+t2  j+t3 k = t_1\;i+t_2\;j+t_3 k=t1​i+t2​j+t3​

所以：

ans=∑i=l1r1∑j=l2r2aijkij=t1∑i=l1r1∑j=l2r2i⋅aij+t2∑i=l1r1∑j=l2r2j⋅aij+t3∑i=l1r1∑j=l2r2aij \begin{aligned} ans &= \sum_{i = l_1}^{r_1}\sum_{j = l_2}^{r_2} a_{ij} k_{ij}\\ &= t_1\sum_{i = l_1}^{r_1}\sum_{j = l_2}^{r_2}i\cdot a_{ij} + t_2\sum_{i = l_1}^{r_1}\sum_{j = l_2}^{r_2}j\cdot a_{ij} + t_3\sum_{i = l_1}^{r_1}\sum_{j = l_2}^{r_2}a_{ij} \end{aligned} ans​=i=l1​∑r1​​j=l2​∑r2​​aij​kij​=t1​i=l1​∑r1​​j=l2​∑r2​​i⋅aij​+t2​i=l1​∑r1​​j=l2​∑r2​​j⋅aij​+t3​i=l1​∑r1​​j=l2​∑r2​​aij​​

容易看到这是三个数组的二维前缀和形式。只需要预处理出这三个二位前缀和数组，就能在 O(1)O(1)O(1) 内完成查询。
