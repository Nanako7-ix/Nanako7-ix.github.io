A - Milya and Two Arrays

Tag

签到

题意

如果一个数组中的任何元素 xxx 至少出现过两次，那么这个数组就叫做好数组。

给你两个长度为 nnn 的好数组 aaa 和 bbb。你可以任意重新排列数组 aaa 中的元素。令 ci=ai+bic_i = a_i + b_ici​=ai​+bi​，问 ccc 中是否可能存在至少 333 个不同的元素

分析

如果 aaa 或者 bbb 中存在 333 种数字的话一定是可以的。如果 aaa 中存在 222 种数字，那么 bbb 种也需要有 222 种数字。

B - Cost of the Array

Tag

模拟，分类讨论

题意

给你一个长度为 nnn 的数组 aaa 和一个偶数 kkk ( 2≤k≤n2 \le k \le n2≤k≤n )，你需要将数组 aaa 分成 kkk 块。

接下来，第 2,4,⋯k22, 4, \cdots \frac{k}{2}2,4,⋯2k​ 块连接成一个数组 bbb。000 被添加到数组 bbb 的末尾。数组 bbb 的代价为使得 bi≠ib_i \neq ibi​=i 的最小下标 iii。求最小代价。

分析

代价的范围是 [1,k2][1, \frac{k}{2}][1,2k​]，考虑暴力找答案。假如已经确定 1,2,⋯ ,i−11, 2, \cdots, i - 11,2,⋯,i−1 都不会成为答案，现在正在判定 iii 是否可以成为答案。

也就是现在 bbb 数组前 i−1i - 1i−1 个数字都已经确定了，第 iii 个数字可以跟在上一块的后面，也可以独立开一块作为新块的开头。

如果跟在上一块的后面，那么需要查看 alst+1a_{lst + 1}alst+1​，其中 lstlstlst 表示上一次选择的位置。如果 i=1i = 1i=1 或者类似于 n=kn = kn=k 的情况是没办法选择跟在上一块后面的。第一个条件很好判断，第二个条件的意思是：选择跟在上一块的后面的次数是有限的，初始为 n−kn - kn−k，判定是否消耗完这些次数即可。

如果选择新开一块，那么一定是在某一个区间中选择一个数，使得它不为 iii。这个区间的左端点是 lst+2lst + 2lst+2，因为 lst+1lst + 1lst+1 放奇数块，然后 lst+2lst + 2lst+2 就是可以开始选择了。右端点取决于剩下还需要放多少个块，假如当前后续还需要放置 xxx 块，那么右端点就是 n−xn - xn−x。

注意到第二种情况左端点取决于 lstlstlst，所以我们尽可能取靠左的数。这个时候中间的奇数块是不会消耗连续放置的次数的。

按照上面的方法写，然后缺什么就维护什么就行了，模拟+分类的恶心感很强。感觉这种题会有更加简单的写法，反正我写红温了。

C - Customer Service

Tag

思维，贪心

题意

给你一个整数 nnn 和 nnn 阶矩阵 aaa。接下来有 nnn 个时间点，每个时间点会发生下面的事情：

时间点编号为 1,2,⋯ ,n1, 2, \cdots, n1,2,⋯,n，fif_ifi​ 初始值为 000。

考虑时间点 xxx，对所有 i=1,2,⋯ ,ni = 1, 2, \cdots, ni=1,2,⋯,n 作：fif_ifi​ 被设为 fi+aixf_i + a_{ix}fi​+aix​。然后你需要选择一个 iii，将 fif_ifi​ 重新设置为 000。

问最后 {f1,f2,⋯ ,fn}\{f_1, f_2, \cdots, f_n\}{f1​,f2​,⋯,fn​} 的 mex 的最大值是多少。

分析

注意到每个数组（指矩阵的行向量）只有最后一次操作是有效的，不妨设每个数组只操作一次且必须操作一次。fff 就是数组的一个后缀和，我们称使得 fif_ifi​ 为 xxx 这件事为数组 iii 获取了数字 xxx。那么容易观察出以下结论：

000 这个数字是无论如何都能取得的，获取 000 这个数需要一个长度为 000 的后缀。获取 111 这个数一定需要一个长度为 111 的后缀，所以获取 222 这个数的数组的后缀一定不能是 111，所以只能是 222。最终我们发现获取 xxx 一定是需要一个全为 111 的长度为 xxx 的后缀。而一个数组能获取数字 xxx 一定可以获取 1,2,⋯ ,x1, 2, \cdots, x1,2,⋯,x

那么对于每一个数组我们记录可以获得的最大的数字，这个值等于后缀中连续的 111 的数量。当我们需要某一个数字 xxx 的时候，我们贪心地选择最大获取值大于等于 xxx 的最小值即可。

D - Graph and Graph

Tag

最短路

题意

给你两个有 nnn 个点的无向连通图，G1G_1G1​ 有 m1m_1m1​ 条边，G2G_2G2​ 有 m2m_2m2​ 条边。每个图都有一个 token，起始位置分别为 s1s_1s1​ 和 s2s_2s2​。你需要执行下面的操作无数次：

假设目前两个图中 token 所在位置分别为 u1u_1u1​ 和 u2u_2u2​，你需要选择 G1G_1G1​ 中与 u1u_1u1​ 相邻的一个点 v1v_1v1​，G2G_2G2​ 中与 u2u_2u2​ 相邻的一个点 v2v_2v2​，将两个图中的 token 分别移动到 v1v_1v1​ 和 v2v_2v2​。然后你的代价会增加 ∣v1−v2∣|v_1 - v_2|∣v1​−v2​∣。

问最小的代价，如果代价为无穷大，输出 −1-1−1。

分析

因为操作时无穷次的，所以如果代价不为无穷大，只能是在 (u,u)(u, u)(u,u) 和 (v,v)(v, v)(v,v) 中反复横跳。我们用最短路求出从初始状态到 (x,y)(x, y)(x,y) 状态所需要的最小代价，然后枚举 u,vu, vu,v 就能求出答案了。

最短路使用dijkstra算法，时间复杂度 O(n2log⁡n)O(n^2\log n)O(n2logn)。
