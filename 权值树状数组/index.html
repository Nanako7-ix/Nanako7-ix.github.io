功能

维护一个集合，操作包括：插入一个数 xxx，删除一个数 xxx，求 xxx 的排名，求集合第 kkk 大，求 xxx 的前驱，求 xxx 的后继

要求

离线，（需要离散化）或者是（值域很小且没有多测）。

核心：树状数组上二分

树状数组上二分是用来求全局第 kkk 大的，功能是在 O(log⁡n)O(\log n)O(logn) 的时间复杂度内求出前缀和大于等于 kkk 的第一个位置。这个操作实际上实在帮我们求出 kth

先放代码

在讲解之前，先来复习一下树状数组中 tr[x] 的含义：

trx=∑i=x−lowbit⁡(x)+1xai tr_x = \sum_{i = x - \operatorname{lowbit}(x) + 1} ^ x a_i trx​=i=x−lowbit(x)+1∑x​ai​

也就是说 tr[x] 表示 [x−lowbit⁡(x)+1,x][x - \operatorname{lowbit}(x) + 1, x][x−lowbit(x)+1,x] 的区间和，该区间的长度为 lowbit⁡(x)\operatorname{lowbit}(x)lowbit(x)。

先抛开树状数组，如果我们需要二分地找到前缀和小于 kkk 的最后一个位置，需要怎么做。先将现有区间分成两部分，如果第一部分的区间和小于 kkk，我们就在第二个区间中讨论，同时 k=k−∑i=lmidaik = k - \sum_{i = l}^{mid} a_ik=k−∑i=lmid​ai​；否则直接进入第一个区间。这种思路在线段树上二分和平衡树求 kth 很常见。

回到代码层面，iii 表示现在考虑的子区间的长度，从 2⌊log⁡2n⌋2^{\lfloor\log_2n\rfloor}2⌊log2​n⌋ 到 111，每次减半（由于 nnn 不一定为 222 的幂次，而为了方便二分，令 iii 为 222 的幂次而不是 nnn。）。x+1x + 1x+1 表示当前区间的左端点。由于 xxx 从 000 开始，而 lowbit⁡(i)\operatorname{lowbit}(i)lowbit(i) 是不断递减的，所以 lowbit⁡(x+i)=lowbit⁡(i)\operatorname{lowbit}(x + i) = \operatorname{lowbit}(i)lowbit(x+i)=lowbit(i)。这保证了 tr[x + i] 的区间为 [x+1,x+i][x + 1, x + i][x+1,x+i]，正好满足了二分区间的要求（再提一下 iii 为子区间的长度）。

现在开始二分操作，如果 tr[x + i] < k（记得先判越界），那么执行走右边的策略，k -= tr[x + i]，同时下次考虑的区间，左端点为 x+i+1x + i + 1x+i+1。根据 xxx 表示考虑区间的左端点，将 xxx 改为 x+ix + ix+i。反之走左边，区间端点不变，kkk 的值不变，所以不需要操作。

这样我们就找到了最后一个前缀和小于 kkk 的位置 xxx，x+1x + 1x+1 就是第一个前缀和大于等于 kkk 的位置，也就是我们要求的 kth

六个操作的完整代码

要插入一个数 xxx，只需要单点修改 a[x]++，即 tr.modify(x, 1)

要删除一个数 xxx，只需要单点修改 a[x]--，即 tr.modify(x, -1)

要求一个数的排名，只需要求小于 xxx 的数的个数，然后加 111。tr.query(1, x - 1) + 1

要求全局第 kkk 大，使用上文的树状数组上二分。

要求 xxx 的前驱，找到小于 xxx 的数的个数 k = tr.query(1, x - 1)，然后求 kth(k)

要求 xxx 的后继，找到小于等于 xxx 的数的个数 k = tr.query(1, x)，然后求 kth(k + 1)
