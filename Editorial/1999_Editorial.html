B - Card Game

statement

Suneet 和斯拉夫玩纸牌游戏。游戏规则如下

  • 每张牌的整数值介于 111 和 101010 之间。
  • 每位玩家收到 222 张牌，这些牌都是面朝下的（所以玩家不知道自己的牌）。
  • 游戏采用回合制，由2个回合组成。在一个回合中，双方随机抽取一张未翻开的牌并翻开。翻开的牌中数字更大的一方获胜。如果数字相等，则无人获胜。
  • 如果一名玩家赢得的回合数最多（即严格意义上大于另一名玩家），则该玩家赢得游戏。如果相等，则无人获胜。

您需要计算 Suneet 有多少种可能最终会成为赢家。

analysis

每人 222 张牌，总共只有两种组合方式: a[0] - b[0] & a[1] - b[1] 或 a[0] - b[1] & a[1] - b[0]。求一下这两种情况能赢的次数，然后输出 2ans2\text{ans}2ans 即可（因为可以互换出牌顺序，即排列）

code

int check (int x, int y) {
    if(x > y) return 1;
    if(x < y) return -1;
    return 0;
}

void solve() {
    // 我尽量写得简洁优雅了
    int a[2], b[2], ans = 0;
    cin >> a[0] >> a[1] >> b[0] >> b[1];
    ans += ((check(a[0], b[0]) + check(a[1], b[1])) > 0);
    ans += ((check(a[0], b[1]) + check(a[1], b[0])) > 0);
    cout << 2 * ans << endl;
}

C - Showering

statement

作为一名计算机科学专业的学生，亚历克斯面临着一个严峻的挑战--洗澡。他努力每天洗澡，但尽管他尽了最大努力，却总是遇到困难。他需要 sss 分钟来洗澡，而一天只有 mmm 分钟!

他已经为一天安排了 nnn 项任务。任务 iii 被表示为一个时间间隔 (li(l_i(li​ , ri)r_i)ri​) ，这意味着亚历克斯很忙，不能在这个时间间隔内（严格来说是在 lil_ili​ 和 rir_iri​ 之间的任意时间点）洗澡。没有两项任务重叠。

给定所有 nnn 时间间隔，亚历克斯当天能洗澡吗？换句话说，亚历克斯是否有一个长度至少为 sss 的空闲时间间隔？

hint

1≤n≤2⋅1051 \leq n \leq 2 \cdot 10^51≤n≤2⋅105 1≤s,m≤1091 \leq s, m \leq 10^91≤s,m≤109 ∑n≤2⋅105\sum n \leq 2 \cdot 10^5∑n≤2⋅105 。

analysis

由于给出的区间没有重合，所以按照时间大小排序，然后每次求相邻两个区间间隔部分的时间 (free time)。将最大值与洗澡需要的时间作比较即可.

code

void solve() {
    int n, s, t, ans = 0, nxt = 0;
    cin >> n >> s >> t;
    vector<pii> a(n + 1);
    FOR(i, 1, n) {
        cin >> a[i].fi >> a[i].se;
        a[i].fi++;
    }
    sort(a.begin() + 1, a.end());
    FOR(i, 1, n) {
        ans = max(ans, a[i].fi - nxt - 1);
        nxt = a[i].se;
    }
    ans = max(ans, t - nxt);
    if(ans >= s) cout << "YES\n";
    else cout << "NO\n";
}

D - Slavic's Exam

statement

存在一个字符串 sss ，它由小写英文字母和可能是零个或多个 ? 组成。

要求斯拉夫人将每个 ? 改成小写英文字母，使字符串 ttt 成为字符串 sss 的子序列。

输出任何这样的字符串，如果没有符合条件的字符串存在，则报告不可能。

hint

1≤∣s∣≤2⋅1051 \leq |s| \leq 2 \cdot 10^51≤∣s∣≤2⋅105，sss 仅由小写英文字母和 ? 组成

1≤∣t∣≤∣s∣1 \leq |t| \leq |s|1≤∣t∣≤∣s∣，ttt 仅由小写英文字母组成

∑∣s∣≤2⋅105\sum|s| \leq 2 \cdot 10^5∑∣s∣≤2⋅105

analysis

由于是子序列，所以一眼贪心，即能选的字符直接选了，不用留着给后面。双指针分别指向两个字符串求解即可。

code

void solve() {
    string s, t;
    cin >> s >> t;
    int n = s.size(), m = t.size();
    for(int i = 0, j = 0; i < n && j < m; ++i) {
        if(s[i] == t[j] || s[i] == '?') {
            if(s[i] == '?') s[i] = t[j];
            ++j;
        }
        if(j == m) {
            cout << "YES\n";
            for(int k = 0; k < n; ++k) {
                if(s[k] == '?') cout << 'a';
                else cout << s[k];
            }
            cout << endl;
            return;
        }
    }
    cout << "NO\n";
}

E - Triple Operations

statement

艾维在黑板上写下了从 lll 到 rrr 的所有整数，inclusive。

在一次运算中，她做了以下操作：

  • 在黑板上选出两个数字 xxx 和 yyy ，擦掉它们，然后在它们的位置上写下数字 3x3x3x 和 ⌊y3⌋\lfloor \frac{y}{3} \rfloor⌊3y​⌋

要使黑板上的所有数字都等于 000 ，艾维最少需要进行多少次运算？我们可以证明这总是可能的。

hint

1≤l<r≤2⋅1051 \leq l < r \leq 2 \cdot 10^51≤l<r≤2⋅105

analysis

可以发现，如果我要将整个序列变成 000，需要先让其中一个数变成 000，然后再让每个数不断取 ⌊x3⌋\lfloor \frac{x}{3} \rfloor⌊3x​⌋。

观察样例: l=1,r=3l = 1, r = 3l=1,r=3. 黑板上的数字为 1,2,31, 2, 31,2,3。将这些数字不断除以 333 变成 000 需要的次数为 1,1,21, 1, 21,1,2。而由于第一步的存在，我们需要额外加上最小的那个次数，即 111，最终答案为 555。

可以得出这样的计算公式:

ans=∑i=lr(⌊log⁡3i⌋+1)+min⁡i=lr(⌊log⁡3i⌋+1)=∑i=lr(⌊log⁡3i⌋+1)+⌊log⁡3l⌋+1\begin{aligned} \text{ans} &= \sum_{i=l}^r (\lfloor \log_3 i\rfloor+1) + \min_{i=l}^r(\lfloor \log_3 i\rfloor+1)\\ &= \sum_{i=l}^r (\lfloor \log_3 i\rfloor+1) + \lfloor \log_3 l\rfloor+1 \end{aligned} ans​=i=l∑r​(⌊log3​i⌋+1)+i=lminr​(⌊log3​i⌋+1)=i=l∑r​(⌊log3​i⌋+1)+⌊log3​l⌋+1​

所以我们可以预处理一个数组表示 ⌊log⁡3x⌋\lfloor \log_3 x\rfloor⌊log3​x⌋，再用前缀和优化求和即可

这个预处理是求 ⌊log⁡αi⌋\lfloor \log_{\alpha} i\rfloor⌊logα​i⌋ 的一个小 Trick，可以记一下

code

预处理

LL lg[MAXN], pre[MAXN];
for(int i = 1; i < MAXN; ++i) {
    lg[i] = lg[i / 3] + 1;
    pre[i] = pre[i - 1] + lg[i];
}

求解

void solve() {
    int l, r;
    cin >> l >> r;
    cout << pre[r] - pre[l - 1] + lg[l]<< endl;
}

F - Expected Median

statement

Arul 有一个。长度为 nnn 的二进制数组 aaa。

他将取该数组长度为 kkk (kkk 为奇数) 的所有子序列，并求出它们的中位数。

所有这些值的和是多少？

由于这个和可能非常大，因此输出它的模数 109+710^9 + 7109+7 。换句话说，打印这个和除以 109+710^9 + 7109+7 的余数。

hint

1≤k≤n≤2⋅1051 \leq k \leq n \leq 2 \cdot 10^51≤k≤n≤2⋅105

kkk 为奇数

analysis

这题相当于是求：在 aaa 中的大小为 kkk 的中位数为 111 的集合的个数。而这样的集合需要 000 的个数小于等于 ⌊k2⌋\lfloor \frac{k}{2} \rfloor⌊2k​⌋

所以计算答案的公式为:

ans≡∑i=0⌊k2⌋−1Ccnt0i⋅Ccnt1k−i(mod109+7)\text{ans} \equiv \sum_{i=0}^{\lfloor \frac{k}{2} \rfloor - 1} C_{\text{cnt0}}^{i}\cdot C_{\text{cnt1}}^{k-i} \pmod {10^9+7} ans≡i=0∑⌊2k​⌋−1​Ccnt0i​⋅Ccnt1k−i​(mod109+7)

求组合数的方法是: 预处理阶乘及其逆元

预处理:

LL f[MAXN], inv[MAXN];

f[0] = 1;
for(int i = 1; i < MAXN; ++i) {
    f[i] = (i * f[i - 1]) % mod;
}
inv[MAXN-1] = fpow(f[MAXN-1], mod - 2);
for(int i = MAXN-2; i >= 0; --i) {
    inv[i] = (inv[i + 1] * (i + 1)) % mod;
}

LL C(int n, int m) {
    if(m > n) return 0;
    return ((f[n] * inv[m]) % mod * inv[n - m]) % mod;
}

求解:

void solve() {
    int n, k, num;
    cin >> n >> k;
    LL ans = 0;
    int cnt0 = 0, cnt1 = 0;
    FOR(i, 1, n) {
        cin >> num;
        if(num) cnt1++;
        else    cnt0++;
    }
    for(int i = 0; i <= k >> 1; ++i) {
        (ans += (C(cnt0, i) * C(cnt1, k - i)) % mod) %= mod;
    }
    cout << ans << endl;
}

G1 - Ruler (easy version)

statement

这是问题的简单版本。两个版本的唯一区别是，在这个版本中，你最多可以进行 10\mathbf{10}10 次查询。

我们有一把暗尺，它缺少一个数字 xxx ( 2≤x≤9992 \leq x \leq 9992≤x≤999 )。当你测量一个长度为 yyy 的物体时，尺子会报告以下数值：

  • 如果是 y<xy < xy<x ，标尺（正确）测量的物体长度为 yyy 。
  • 如果为 y≥xy \geq xy≥x ，则标尺错误地测量出物体的长度为 y+1y+1y+1 。

您需要找到 xxx 的值。为此，您可以进行以下形式的查询：

  • ? a b\texttt{?}~a~b? a b - 作为回应，我们将用尺子测量一个 a×ba \times ba×b 矩形的边长，并将结果相乘，向您报告测量出的矩形面积。例如，如果 x=4x=4x=4 和您查询的矩形是 3×53 \times 53×5 ，我们将测量其边长为 3×63 \times 63×6 ，并向您报告 181818 。

求 xxx 的值。您最多可以查询 10\mathbf{10}10 次。

analysis

这个问题有点像数字炸弹，每次询问一个数字，法官回答偏大、偏小或相等。想要尽快确定出这个数字，我们每次都要查询区间中点的数。

注意到 xxx 的范围是 [2,999][2, 999][2,999]，约等于 100010001000，我们容易想到 210≈10002^{10} \approx 1000210≈1000，进而可以确定这个方法是对的。

code

void solve() {
    // 与二分答案不同
    // 这里 l 永远能被正确读数
    // 而 r 永远不能被正确读数
    int l = 1, r = 999, ans;
    while(l + 1 != r) {
        int m = l + r >> 1;
        cout << '?' << ' ' << l << ' ' << m << endl;
        fflush(stdout);
        cin >> ans;
        if(ans == l * m) {
            l = m;
        } else r = m;
    }
    cout << "! " << r << endl;
    fflush(stdout);
}

G2 - Ruler (hard version)

statement

同上，只是查询的次数限制到了 7\mathbf{7}7 次

analysis

如果继续用二分的方法，777 次肯定是不够用。而上面的解法本质上只用到了其中一个点，因为 lll 不需要查询都知道是能够正常读数的，所以我们考虑三分。

code

void solve() {
    int l = 1, r = 999, ans;
    // 依旧是 l 正常, r 不正常
    while(l + 1 != r) {
        int m1 = l + (r - l) / 3;
        int m2 = m1 + (r - l) / 3;
        if(r - l == 2) m2++;
        cout << '?' << ' ' << m1 << ' ' << m2 << endl;
        fflush(stdout);
        cin >> ans;
        if(ans == m1 * m2) {
            l = m2;
        } else if(ans == m1 * (m2 + 1)) {
            l = m1, r = m2;
        } else {
            r = m1;
        }
    }
    cout << "! " << r << endl;
    fflush(stdout);
}

第一次写三分，应该是这样的，注意当 r - l == 2 的时候，要放一个到中间的位置。
