01背包

其实这个是我在刷了一部分题后重新写的，我长话短说，感觉不是很适合入门但是可以看看

问题导入

现在在你面前有 nnn 个物品，每个物品有两个属性：价值 v，与体积 w。你有一个大小为 mmm 的背包，求你能获得的最大的价值

analysis

现在直接令 dp[i] 表示大小为 iii 的背包能获取的最大的价值，显然我们要求的是 dp[m].

我们扫一遍所有的物品，对于每一个物品，我们只有两个决策，装进背包和不装。这两种决策都可能对已有的最优解们 dp[i] 产生影响，于是我们可以得到以下的两种可能。

  • dp[i] <- dp[i] 不装进背包, 此时的变化就是没有变化.
  • dp[i] <- dp[i - w] + v 装进背包，此时背包的剩余容量为 i−wi - wi−w，不难想到这种情况下可以获取的最大价值为 dp[i - w] + v.

所以就能得到：dp[i] = max(dp[i], dp[i - w] + v)，也就是第一种决策带来的变化和第二种决策带来的变化的最大值。

能改变一个就能改变整个 dp[0 ~ m]。注意我们要从大到小改变 dp[i]。原因是：dp[i - w] 已经计算过了，说明这个背包有可能是装过该物品的了，此时就不能执行第二个决策，那么上面的转移方程就不对了（每个物品只能放一遍）

做法就显而易见了，对于每个物品，都改变一次 dp[0 ~ m]。最后得到的 dp[m] 就是我想要的答案

Begin

先看最经典的01背包：有 nnn 件物品。第 iii 件物品的体积和价值分别为 wiw_iwi​，viv_ivi​. 已知背包容量为 VVV，求背包中最多能装入多少价值的物品。

都来看这篇文章了肯定是已经知道这个问题怎么在 O(nV)O(nV)O(nV) 的时间复杂度内解决。下面直接来说结论，有关01背包的一些我认为正确的东西。

  • 需要考虑 nnn 个子决策点（物品），每个子决策点存在 O(1)O(1)O(1) 数量的决策方案
  • 每个决策点（物品）涉及到两个维度，且显示出此消彼长的特性（物品有两个维度指物品会产生两方面的影响，不等于物品有两个属性）

感觉这两个点比较关键，就是首先考虑决策点是否存在这两个特点，存在就可以考虑01背包

这个时候的做法就是遍历所有的决策点，考虑所有决策对于目前已有的 dp 数组的影响（状态转移）

这个 dp 数组一般视情况而定，一般表示决策的两个维度

End

例题时间

我觉得已经是比较清晰了，下面来一道例题，也是上面引述的文章的题目

题解

analysis

一共存在 nnn 个决策点，每次垃圾丢下来都是一个决策。
决策方案为 222 个：吃掉，堆起来
每个决策存在两个维度：生命值，高度

我们将这两个维度分别对应 dp 数组的下标与值
（注意这里这里不用太过于在意区别这两个维度哪个是贡献，哪个是代价，因为这两种写法可以换着来写）

solution

状态的划分：dp[i] 表示如果这头牛在 t=it = it=i 时死亡，那么它最终爬升到的高度最大值。

每次决策的决策范围 [a,b][a, b][a,b]，aaa 表示：想要吃到这块垃圾至少要活到第 iii 秒，bbb 表示：通过前面的垃圾，最多能活到多少秒。

状态转移方程：
选择吃掉，dp[j + 饱食度] <- dp[j]，也就是说:活的更久了，但是没有提升高度
选择堆起来，dp[j] += 高度 很显然就是直接在目前的生命值下直接堆高度。

然后这题就算是做完了，因为得到 dp[] 之后就可以通过 dp[] 来找答案了，这里不多赘述。

方案数dp

普通方案数
nnn 个物品，大小为 mmm 的背包，求 `dp[i]`，`dp[i]` 表示恰好装满大小为 iii 的背包的方案数。
不考虑某件物品的方案数
nnn 个物品，大小为 mmm 的背包，求 `f[i][j]`，`f[i][j]` 表示**不装入**第 iii 件物品时，恰好装满大小为 jjj 的背包的方案数

先正常跑一遍方案数的dp，求出 dp[]，dp[i] 表示恰好装满大小为 iii 的背包的方案数。

然后依据 dp[] 求出 f[][]。具体做法如下

  • 考虑 f[i][] 这个数组，以下称其为 g[]，我们有以下的转移过程
  • 如果 j<wj < wj<w，则 g[j] = dp[j]，因为 dp[j] 的所有方案里面一定不存在含这个物品的方案（因为根本放不下）
  • 如果 j≥wj \ge wj≥w，则 g[j] = dp[j] - g[j - w]。要求含有这个物品的方案数，我首先会在背包里面放入这个物品，然后装满剩下 j−wj - wj−w 的背包，这一部分的方案数为 g[j - w]，根据乘法原则，这种情况的方案数量为 1 * g[j - w]。然后再用所有方案（数）减去含有该物品的方案（数），最后算出来的是不含该物品的方案（数），即 dp[j] - g[j - w]

这样可以使得装不满的背包显示为 inf，从而在计算 ans 的时候被舍去

二维01背包

一个物品的属性维度如果不只是有两个怎么办？那就把二维视作一维就行了

下面来看一道例题

总所周知一个物品能不能装进背包不仅要看物品的体积，还要看他的重量

现在有 nnn 个物品，物品有质量，体积，价值；
求在一个体积限制为 VVV，质量限制为 MMM 的背包中所能装入的物品最大价值

analysis

注意到体积和质量其实都是差不多的概念，我们可以把它当成一个代价维度，类型是 pair<int, int>，然后我们根据 pii 作为索引去查找一个背包的最大贡献，显然就有解法了。但是用 pii 作索引太傻逼了，所以我们用二维数组

warning

如果发现时间复杂度不对，请检查是否能将两个代价维度化作一个。一般这种情况下，两个代价维度有一定联系，例如利润与成本，可以合并。

常见优化：互换键-值

我们都知道背包dp的时间复杂度为 O(nV)O(nV)O(nV)。现在数据范围: 物品数量 300030003000，物品大小 10610^6106，物品价值 111（虽然这样可以用贪心，但是你不准用qaq）

如果按照大小作为键，价值作为值，那么这个 dp[] 要开到 3⋅1093\cdot 10^93⋅109 的大小，而 dp[x] 最多不过 300030003000。这个时候我们互换键与值，让 dp[x] 表示价值为 xxx 需要的大小，这样就合理多了。

当键的值域过大且值的值域较小的时候，可以考虑互换键值的优化

分组背包

问题导入

现在在你面前有 nnn 个物品，每个物品有两个属性：价值 v，与体积 w。你有一个大小为 mmm 的背包，求你能获得的最大的价值。

特殊限制: 这 nnn 个物品被分成 kkk 组，每组物品只能选取一个

analysis

有了01背包的基础，我们或许有点思路。这里直接给代码，自己尝试看懂。

分组背包的本质

观察问题发现，问题的主干与01背包完全一致，只是多了一个特殊限制条件。这个条件可以被总结为: 存在一部分的决策互斥，通俗的说就是只能选一个。

决策点为: 第 iii 组第 jjj 个物品的选择.

这样符合01背包的条件:

  • 存在若干个决策点，每个决策点存在 O(1)O(1)O(1) 种选择
  • 每个决策涉及两个维度，二者呈此消彼长的关系

同时符合分组背包的特殊限制

  • 存在一部分的决策互斥

树形背包

前置知识: 树形dp

给定一棵树，选择 u 必须先选择 fa，每个点有其贡献和代价，求大小为 m 的背包能获得的最大价值。

设 dp[i][j] 表示以 i 为根的子树，大小为 j 的背包所能获得的最大的价值。

显然以 v 子树有一系列的选择，具体说就是有若干的 [v, dp[v][?]] 组, 供 u 选择，而对 dp[u] 产生贡献的只能有一种（i 的最终取值唯一）。于是我们得出这个结论:

在 u 处做的决策中，存在一部分决策互斥，于是可以使用分组背包

解释一下这个枚举顺序

j 和 k 的枚举顺序不能反。如果反过来，就对于 k = 1 枚举了所有大小的 dp[u]，再对 k = 2 枚举所有的 dp[u]，就会违背 01背包的原则。在这里指的是，同时选取一棵子树不同大小的背包多次（dp[v][x] 和 dp[v][y]）。

时间复杂度: O(nm2)O(nm^2)O(nm2)

一个非常值得注意的点

要有默认的方案选择，因为大部分情况下，默认选择贡献为0，代价为0的决策，所以关注不到这点。

分组背包中决策是要枚举的，很有可能导致做出 000 个决策。一定要想清楚决策是什么，默认决策是什么.

题目link

大意是要切出一个大小为 PPP 的子树，求最少要断几条边

dp[u][j] 以 u 为根的子树，贡献 j 个点，至少需要且多少条边.

这里默认选择 v 贡献 000 个点，而贡献 000 个点需要 111 条边，所以每个大小的背包都先加 111，再进行DP。
