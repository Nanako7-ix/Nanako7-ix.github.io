A - Twice

题意

给你一个序列，你需要选择两个之前没有选择过的相同的数，将分数+1，求最高分数。

分析

记录每一个数出现的次数，分数就是每个数的出现次数除以 222 下取整。

B - Intercepted Inputs

题意

给你一个序列，你需要在这个序列中找到两个数 a,ba, ba,b 使得 ab=n−2ab = n - 2ab=n−2

分析

一般来说像这种找两个数的题目（例如确定一个区间），都不会让你枚举两个数。一般是枚举第一个数然后再搞一搞第二个数。所以枚举第一个数，然后判断 n−2x\frac{n-2}{x}xn−2​ 是否出现过就行了。由于保证答案一定存在，所以不需要判 a=ba = ba=b 的时候是否出现两次 aaa

C - Superultra’s Favorite Permutation

题意

你需要构造一个排列，使得任意两个相邻的数的和都是合数。

分析

容易想到让两个数的和为偶数，这样是更容易操作的。因为奇数+奇数=偶数，偶数+偶数=偶数。所以把奇数放一边，偶数放一边。找一个奇数和一个偶数，使他们的和为合数，然后把这两个数放在奇偶交界处。直接枚举他们的和即可。

D - Sharky Surfing

题意

你有一个初始跳跃能力 k0=1k_0 = 1k0​=1，表示你可以从 xxx 跳跃到 [x,x+k][x, x + k][x,x+k] 之间的任意一个位置。而途中存在 nnn 个障碍，用一个区间 [l,r][l, r][l,r] 表示。你不能跳到这个区间内的任何一个位置。地图上有 mmm 个增益道具，每个增益道具可以为你提升 viv_ivi​ 的跳跃能力。你需要尽可能少使用增益道具的情况下从 111 到达终点 LLL。问你至少使用多少个增益道具。

分析

这题太简单了所以没有分析，只有我对反悔贪心的一些小讨论，可以直接看代码

其实这题是一个反悔贪心，想不到吧这么简单的题目居然是反悔贪心这种听起来这个高级的东西。介绍一下返回贪心的逻辑：你会遇到许多的决策，你的决策可能现在看不出来是好的决策还是不好的决策。所以你先做目前最优的决策，并且将你决策的反悔方案存起来，然后如果你在以后发现你当初的决策是错误的，那么那掏出你的反悔决策，就可以实现反悔了。这么说有点抽象，具体看题目。

假如我遇到了一个增益道具，由于我想要尽可能少使用增益道具，那么现在采取的策略就是不使用。反悔策略当然就是增益道具了，所以我们将增益效果存起来，如果以后遇到困难，我再把这个增益效果加到我的能力值上。这个反悔的过程也是贪心地，也就是我一定是选择增益效果最强的道具来使用。这个单调性就是用优先队列维护就行。

其实反悔贪心最经典的一个例子就是网络流，建立反向边就是一个反悔决策。每次都是有路就走，然后反向边加流量。每次走反向边相当于给原来的边做反悔。顺便来道例题，不过难度比这题要难。

E - Kachina’s Favorite Binary String

题意

交互题，需要构造一个 010101 字符串。你可以询问一个区间 [l,r][l, r][l,r]，然后会告诉你这个区间内 010101 子序列的个数。

分析

考虑长度为 考虑查询区间列 [1,n],[2,n]⋯[n−1,n][1, n], [2, n]\cdots[n - 1, n][1,n],[2,n]⋯[n−1,n]。如果查询到子序列的数量减少了，那么说明左侧有一个 000，如果不变，说明是 111。如果减少到了 000，那最后的一部分就判断不了是 110011001100 还是 100010001000 之类的了。所以如果检测到了 000，那么上一次查询一定答案不是 000，假设上一次查询的区间为 [l,r][l, r][l,r]，那么接下来查询区间列 [l,n−1],[l,n−2],⋯ ,[l,l+1][l, n - 1], [l, n - 2], \cdots, [l, l + 1][l,n−1],[l,n−2],⋯,[l,l+1]，如果再次检测到 000，说明中间的部分全是 000，否则就按照先前的策略判断 000 或 111 即可。这样就能确定整个字符串了。注意到，总共的查询次数是符合要求的。

特判：如果第一次查询，即查询区间为 [1,n][1, n][1,n] 时，返回的结果是 000。由于没有上一次查询，所以无法确定结果。

F - Ardent Flames

题意

水平线上存在 nnn 个敌人，你可以在任意一个位置放置角色，让角色攻击敌人，使得最终击败不少于 kkk 个敌人，问在这个基础上至少需要攻击多少次。如果不可能满足击败不少于 kkk 个敌人的要求，输出 −1-1−1。

角色的攻击机制：假如角色所在位置为 ppp，那么处在 xxx 的敌人收到的伤害为 max⁡(d−∣p−x∣,0)\max(d - \lvert p - x \rvert, 0)max(d−∣p−x∣,0)

分析

其实这题的二分答案还是挺容易看出来的，现在问你怎么check。

也就是如何判断攻击次数为 mmm 的时候能击杀的敌人数量大于等于 kkk。考虑一个敌人在 mmm 次攻击内死亡的条件，容易发现角色必须位于区间 [p−⌈hpm⌉,p+⌈hpm⌉][p - \lceil\frac{hp}{m}\rceil, p + \lceil\frac{hp}{m}\rceil][p−⌈mhp​⌉,p+⌈mhp​⌉] 内。于是问题就变成了：判断是否存在一个点，使得能被至少 kkk 个区间覆盖。这是一个典得不能再典的问题。

解法是将区间端点离散化，记录每个点被区间覆盖的次数。对于每个区间，进行一次区间修改，然后查询最大值。这个东西显然线段树能做，但是可以优化成差分，差分需要记得判断左端点不能大于右端点。

时间复杂度 O(nlog⁡Vlog⁡n)O(n \log V \log n)O(nlogVlogn)，VVV 表示 hp 的值域。

G - Natlan Exploring

题意

给你一个序列，如果 gcd⁡(ai,aj)≠0,i<j\gcd(a_i, a_j) \neq 0, i < jgcd(ai​,aj​)=0,i<j，那么可以从 iii 向 jjj 连一条边。问从 111 到达 nnn 的路径方案数是多少。

分析

考虑暴力，dpidp_idpi​ 表示到达位置 iii 的方案数，转移方程也非常简单，就是把小于 iii 的所有的 jjj 遍历一遍，如果 gcd⁡(ai,aj)\gcd(a_i, a_j)gcd(ai​,aj​) 就加上它的贡献。这样是 O(n2log⁡V)O(n^2\log V)O(n2logV) 的。下面考虑优化。

考虑 aia_iai​ 做一个质因数分解，Πi=1npiti\Pi_{i = 1}^n p_i^{t_i}Πi=1n​piti​​。显然 gcd⁡(aj,ai)\gcd(a_j, a_i)gcd(aj​,ai​) 也一定是这个形式的。我们枚举 gcd⁡(aj,ai)\gcd(a_j, a_i)gcd(aj​,ai​) 的质因数分解情况，然后再利用容斥原理做贡献。例如 121212 这个数做质因数分解就是 p1=2,p2=3p_1 = 2, p_2 = 3p1​=2,p2​=3，那么存在 222 这个质因子的数能产生贡献，存在 333 这个质因子的数也能产生贡献。我们让 dpidp_idpi​ 表示从 111 到含有 iii 这个质因子的数的路径数。很容易想到如果我有一个数 666，那么它将产生两次贡献，所以容斥一下就解决了。

质因数分解：O(很小)O(很小)O(很小)，枚举gcd的质因子情况 O(2max⁡(ω))O(2^{\max(\omega)})O(2max(ω)) 其中 max⁡(ω)\max(\omega)max(ω) 为 777，线性筛预处理 O(n)O(n)O(n)。总时间复杂度 O(n⋅(2max⁡(ω)+很小))O(n\cdot(2^{\max(\omega)} + 很小))O(n⋅(2max(ω)+很小))
