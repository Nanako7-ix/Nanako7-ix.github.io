D - Swap to Gather

Tag

贪心

题意

给你一个 010101 字符串，保证至少包含一个 111，你可以执行操作：交换相邻两个字符。问至少需要交换多少次使得字符串中的 111 连续。字符串长度 5⋅1055\cdot 10^55⋅105。

分析

有点像是一个挤气泡的操作，把 000 往 111 少的那边交换。如何判断左边的 111 多还是右边的 111 多：对 111 出现的位置开一个桶，然后在桶上二分即可。时间复杂度 O(nlog⁡n)O(n\log n)O(nlogn)。

E - GCD of Subset

暴力、数学

题意

给你一个数组，对于每个 i∈[1,n]i \in [1, n]i∈[1,n] 计算：在数组 aaa 中选择包含 aia_iai​ 在内的 kkk 个数，计算他们的 gcdgcdgcd，求 gcdgcdgcd 的最大值。数组长度 1.2⋅1061.2\cdot 10^61.2⋅106，值域 10610^6106。

分析

有一个简单但是重要的结论，答案一定是 aia_iai​ 的因数。所以逆序枚举 aia_iai​ 的因子 xxx，然后判断数组中 xxx 的倍数出现的次数是否大于等于 kkk，如果满足说明 xxx 为答案。根据下表可知时间复杂度为 O(240n)O(240n)O(240n)，预处理得到 10610^6106 内所有数的因数的时间复杂度为 O(Nlog⁡N)O(N\log N)O(NlogN)。总时间复杂度 O(Nlog⁡N+240n)O(N\log N + 240n)O(NlogN+240n)。

F - Prefix LIS Query

Tag

LIS、数据结构

题意

给你一个数组 aaa，给你 qqq 个查询。每次查询给定两个整数 r,xr, xr,x，问你考虑前 rrr 个数且值域不超过 xxx 的最长上升子序列的长度。n,qn, qn,q 同阶，大小 2⋅1052\cdot 10^52⋅105。aia_iai​ 值域 10910^9109，r,xr, xr,x 合法。

分析

一个单调递增的子序列，要求值域不超过 xxx，实际上也就只是要求最后一个数不超过 xxx。考虑以 a1,a2,⋯ ,ara_1, a_2, \cdots, a_ra1​,a2​,⋯,ar​ 结尾的LIS，如果 ai≤xa_i \leq xai​≤x，那么它的 LIS 的长度就可以对答案产生贡献。

我们离线所有查询，按 rrr 从小到大的顺序查询。使用数据结构维护LIS末尾为 ai,i≤ra_i, i \leq rai​,i≤r 的LIS长度，然后查询 [1,x][1, x][1,x] 的最大值即可。

以 aia_iai​ 结尾的LIS长度不可能不会吧，dp一下就能在 O(nlog⁡n)O(n\log n)O(nlogn) 的时间复杂度内预处理出来。然后维护LIS末尾为 ai,i≤ra_i, i \leq rai​,i≤r 的LIS长度，这个可以使用离散化+树状数组做。我懒得离散化直接动态开点线段树做了，反正可以用板子。
